---
description: 按计划分批执行（默认前 3 任务）、校验并在批次间回报。
---

## 用户输入

```text
$ARGUMENTS
```

可用于指定沟通语言（默认中文）或传入计划路径等上下文。

## 目标

加载并批判性审查计划后再执行：按批次（默认首批 3 个任务）将任务置为 in_progress → 完成，严格照计划步骤和验证要求，批次结束汇报进展与验证结果，再等反馈后继续。

## 执行步骤

1) **开场与语言**
   - 宣告："I'm implementing this plan."
   - 沟通语言遵循传入语言；缺失时使用中文。

2) **加载与审查计划**
   - 读取指定的计划文件。
   - 批判性审查，若发现缺口/疑问/阻塞，先向用户提出后再动手。
   - 无问题则创建 TodoWrite（若工作流需要）并继续。

3) **批次执行（默认前 3 任务）**
   - 对批次内每个任务：
     - 标记状态为 in_progress。
     - 严格按计划子步骤执行。
     - 按计划运行验证（测试/脚本等），记录输出。
     - 成功后标记 completed。

4) **批次汇报**
   - 批次结束后说明：已完成的任务与关键实现、验证输出。
   - 结尾询问："Ready for feedback."，等待下一步指示。

5) **继续或收尾**
   - 根据反馈调整或继续下一批任务，重复执行-汇报流程。
   - 所有任务完成且验证通过后宣布："I'm completed this implementation."

## 停止/回退原则

- 若遇依赖缺失、测试失败、指令不清、计划缺口或重复验证失败：立即停止执行并向用户求助。
- 若计划更新或思路需调整，回到“加载与审查计划”重新确认。
- 不要跳步或跳过验证；不清楚时优先澄清。
