## 用户输入

无需额外输入；命令依赖当前会话历史。

## 前置条件

- 会话中存在 `/spec:brainstorm` 调用；若未找到，明确告知并退出，不写文件。

## 目标

定位最近的脑暴会话，将其后的用户+助手消息提炼为结构化设计文档，命名并保存到 `docs/designs/YYYY-MM-DD-<slug>.md`（slug 为 3-5 个小写连字符词）。

## 执行步骤

1) **定位会话**
   - 在会话历史中查找 `/spec:brainstorm` 调用。
   - 收集其后的所有用户与助手消息；若不存在则提示用户并停止。

2) **推断主题与准备元信息**
   - 基于收集内容推断主特性/主题，生成 slug（3-5 词，小写、连字符）。
   - 获取当前日期 `YYYY-MM-DD`，构造目标文件 `docs/designs/YYYY-MM-DD-<slug>.md`。
   - 若文件已存在，追加如 `-1`、`-2` 或短时间戳后缀避免覆盖。
   - 若 `docs/designs` 目录不存在，则创建。

3) **撰写文档**
   - 严格使用以下结构：
     ```
     # [Feature Name]

     **Date:** YYYY-MM-DD

     ## Context
     ...

     ## Discussion
     ...

     ## Approach
     ...

     ## Architecture
     ...
     ```
   - 内容全部来源于脑暴后的对话：Context 写初始动机，Discussion 写关键问题/取舍/备选方案，Approach 写最终方向，Architecture 写技术细节/流程（若有）。
   - 不包含原始聊天记录，仅保留提炼后的结论。

4) **落盘与回报**
   - 写入文件并保持同名文件的去重规则。
   - 最后输出写入的完整文件路径。

## 注意事项

- 响应语言保持与脑暴时一致（无法识别时默认中文）。
- 若未检测到脑暴会话，务必明确说明并避免任何写入。
