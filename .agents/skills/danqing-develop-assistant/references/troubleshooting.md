# 常见问题速查（agent 内部参考）

| 用户的描述 | 内部原因 | 解决方式 | 向用户的说法 |
|---|---|---|---|
| "工具不好使" / 功能没反应 | MCP 服务未生效 | 检查配置和重启 | "开发服务需要重新连接一下，请按我的提示操作" |
| "页面打不开" | hosts 未配置 | 引导配置域名 | "需要做一个小设置让浏览器能找到您的页面" |
| "页面打不开"（地址被跳转成 https） | 浏览器缓存了 HSTS 策略（whistle 等转发工具曾以 HTTPS 代理该域名） | 见下方「HTTPS 自动跳转修复」 | "浏览器记住了之前的安全设置，我来帮您清除" |
| "数据存不了" | 数据库权限问题 | 授权或清除缓存 | "数据服务遇到小问题，我来修复" |
| "发布失败" / "太大了" | 包含大图片/文件 | 上传到 bucket | "项目中有些大文件需要单独处理，我来搞定" |
| "页面发布后样式不对" | 构建路径问题 | 检查 base 路径配置 | "我来检查一下发布配置" |
| "怎么邀请同事一起开发" | 需要邀请码 | 调用 invite_member | "我生成一个邀请码，发给对方就可以加入了" |

## HTTPS 自动跳转修复

**现象**：用户访问 `http://danqing-local.danlu.netease.com:端口` 时，浏览器自动跳转成 `https://...`，导致页面无法加载。

**原因**：用户本地曾使用 whistle、Charles 等转发工具以 HTTPS 代理过该域名，浏览器缓存了 HSTS（HTTP Strict Transport Security）策略，之后会强制将该域名的所有 HTTP 请求升级为 HTTPS。

**修复方法**（引导用户操作，用大白话描述）：

> 您的浏览器之前记住了一个安全设置，导致页面无法正常加载。请按以下步骤清除：
>
> **Chrome 浏览器**：
> 1. 在地址栏输入 `chrome://net-internals/#hsts`，按回车
> 2. 找到页面下方的 **Delete domain security policies** 区域
> 3. 在输入框中输入 `danqing-local.danlu.netease.com`
> 4. 点击 **Delete** 按钮
> 5. 关闭该标签页，重新访问页面即可
>
> **Edge 浏览器**：
> 1. 在地址栏输入 `edge://net-internals/#hsts`，按回车
> 2. 其余步骤同上
>
> **Firefox 浏览器**：
> 1. 关闭所有页面，然后清除该网站的浏览数据
> 2. 或者：在历史记录中找到该域名，右键选择「忘记此站点」

如果用户不愿意操作或操作后仍有问题，可以建议使用**无痕/隐私模式**打开页面（无痕模式不受 HSTS 缓存影响）。
