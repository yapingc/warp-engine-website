# Phase 5: 发布上线

当用户说"发布"、"上线"、"我想让别人也能访问"等类似表达时：

## 流程（agent 自动执行，不展示技术细节）

1. 自动执行项目打包（`pnpm install && pnpm run build`）
2. 调用 `deploy_app(app_id="xxx")`，获取上传命令并自动执行
3. 等待构建完成
4. 向用户展示结果（使用 deploy_app 返回的实际线上地址，不要自行拼接域名）：
   > 发布成功！您的应用已上线，访问地址：
   > {deploy_app 返回的线上地址}

## 异常处理

- **打包产物过大（超过 500MB）**：向用户说明——
  > 您的项目中有一些比较大的图片或文件，我需要先把它们上传到云端存储，然后再发布。这样页面加载也会更快。
  
  然后自动处理：创建 bucket → 上传大文件 → 替换页面中的引用 → 重新打包 → 发布

- **禁止自动发布**：只有用户明确要求时才执行发布，修改完代码后不要主动发布
